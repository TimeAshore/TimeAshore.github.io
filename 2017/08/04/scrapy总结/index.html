scrapy问题汇总

发表于 2017-08-04 | 分类于 爬虫 | | 阅读次数
字数统计 745 | 阅读时长 3

Scrapy 总结：

1，是否需要登录，需要就要设置cookie （Request,FormRequest）
2，是否IP被封锁。 （加中间件，切换代理IP）

3，Headers 中的User-Agent问题，scrapy默认有headers,但是这个headers跟浏览器的headers是有区别的，有的网站会检查headers，如果是浏览器的headers就允许通过，是机器人的或者说是爬虫的就拒绝，所以在setting.py中设置一个headers就ok了 （加浏览器User-Agent中间件，切换User-Agent）
4，禁用爬虫的Robots.txt （有的网站检测到Robots.txt就封，设为Flase不遵循）
ROBOTSTXT_OBEY = False
5，出现IndexError: list index out of range 错误时：仔细看一看网页的代码,很可能是选择器选择的位置错误。

各网站处理状态
1、智联简历。缺少账号，每个账号大概能爬500-800份，之后会出现验证码，账号多了来回切换解决问题。
2、文书网。可以爬，会被封IP，爬取的西刺免费ip能用的有几十个，一共能爬下来文书就500个左右，猜测是免费IP的质量不好。
3、万行教师人才网。可以爬（静态网页，登陆上以后找到cookie,爬的时候带上去），但爬几万份简历后程序自己就停了，scrapy状态是（finished），晚上可以爬十几万，白天爬2w就左右就停了。（2017.06.25补充：万行教师人才网数据已经全部爬完，之前会无辜停止的原因是爬取速度过快，网站的服务器压力太大，设置DOWNLOAD_DELAY=0.2完美过关）
4、拉钩网。 ajax加载数据，返回json文件渲染进网页，构造json文件的URL就可以了（需要带cookie,header, POST请求）。
5、简书首页、七日热门、三十日热门，规律一样，第二页url由第一页文章的id组成（第一页是静态页面，其他都需要抓包），第三页url由第一页id加第二页id组成，以此类推。爬虫在十页左右无辜停止，原因是十几页的时候url就很长很长了，而默认的爬虫url长度有限，学到新的知识：URLLENGTH_LIMIT = 20000 #允许爬取的URL的最大长度， 设置这个长度就可以了
6、中国服装人才网，静态，个人页面的代码在一个post请求里，比较隐蔽，页码递增直接爬就行 /爬完了
7、北京教育人才网，要登陆。所在地，所学专业是从js文件里加载，本方法把各个城市、专业都写到python字典里，从页面获取key,字典里得到value，数据不多，两万三千，其中有几千个是空数据。
8、注册建筑师类。网站服务器很差，设置一秒延迟也很勉强。一级注册建筑师已爬完，一级二级临时执业人员已爬完，二级注册建筑师正在爬。
